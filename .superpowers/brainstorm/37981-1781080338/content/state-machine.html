State Machine & Phase Logic

Full lifecycle from blank canvas to complete formula. Does the branching look right?

Phase Diagram
SELECTING
Both zones empty. Tray fully interactive. Crossover button disabled.
↓ DROP_ELEMENT (either side)
DEDUCING_CHARGE (per zone — cation first, then anion)
Dropped zone expands inline. Other zone locked (opacity 40%, no drop).
Three sub-paths based on element type:
📗 Regular (s/p block, non-transition)
Show lose/gain choice → count choice → validate
Wrong: shake + wrongCount++. At wrongCount ≥ 2: show valence hint.
Correct: electron animation → symbol transforms to ion → IONIZED
📙 Transition metal (d-block: group 3–12)
Show charge picker (all valid oxidation states from WASM oxidation_states[])
Student picks → electron count auto-filled → confirm → IONIZED
📕 Polyatomic ion
Show read-only charge callout (e.g. "OH⁻ carries −1 charge")
Student taps "Got it" → immediately IONIZED (no wrong-answer path)
↓ Both zones IONIZED
READY_TO_CROSS
Both ions shown. Bridge column pulses. "Cross Over →" button enabled.
↓ TRIGGER_CROSSOVER
ANIMATING_CROSSOVER
4-frame Framer Motion sequence (locked, no interaction):
① Charge signs (±) fade out → bare integers remain
② Integers animate diagonally across bridge (X path)
③ If polyatomic target AND subscript > 1 → brackets animate in
④ GCD check → if GCD > 1: divide animation (e.g. ÷2) → simplified formula
↓ CROSSOVER_COMPLETE
COMPLETE
Final formula displayed in bridge. Reset button shown. No further interaction.
State Schema (TypeScript)
interface IonicCanvasState {
  canvasPhase: 'SELECTING' | 'DEDUCING_CHARGE' | 'READY_TO_CROSS'
            | 'ANIMATING_CROSSOVER' | 'COMPLETE';
  activeDeductionSide: 'cation' | 'anion' | null;
  cation: ZoneState | null;
  anion:  ZoneState | null;
}

interface ZoneState {
  symbol:          string;
  isPolyatomic:    boolean;
  isTransition:    boolean;
  valenceElectrons: number;       // from WASM electron_configuration
  oxidationStates: number[];      // from WASM — used for TM picker
  derivedCharge:   number | null; // set after successful deduction
  wrongCount:      number;        // resets on correct answer
  status: 'NEUTRAL' | 'DEDUCING' | 'IONIZED';
}
