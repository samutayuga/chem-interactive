Architecture Overview

Component tree and data flow. Does this structure look right?

Component Tree
App
├── WasmProvider // loads PeriodicTable.load() once, exposes via context
└── IonicCanvasProvider // useReducer: IonicCanvasState + dispatch
└── IonicCanvas // root layout — bridge arrangement
├── ElementTray // scrollable neutral atoms + polyatomic tab
└── ElementToken × N // draggable chip: symbol, name, atomic number
├── DropZone // side=cation|anion, reads phase from context
└── DeductionPanel // inline expansion — shown when zone has NEUTRAL element
├── RegularDeduction // lose/gain + count buttons
├── TransitionMetalPicker // charge selector for d-block
└── PolyatomicConfirm // read-only charge callout + Got it
├── BridgeColumn // centre column: ⇌ icon, crossover button, formula
└── DropZone // anion side (mirror of cation)
Data Flow
ElementToken drag → DROP_ELEMENT action → reducer sets zone to NEUTRAL
DeductionPanel answer → SUBMIT_DEDUCTION → reducer validates → sets zone to IONIZED
BridgeColumn button → TRIGGER_CROSSOVER → phase → ANIMATING_CROSSOVER
CrossoverAnimator completes → CROSSOVER_COMPLETE → phase → COMPLETE
WasmProvider → useWasm() hook → any component needing element data
Reducer Actions
DROP_ELEMENT { side, symbol }
SUBMIT_DEDUCTION { side, loseOrGain, count } // regular
PICK_TM_CHARGE { side, charge } // transition metal
CONFIRM_POLYATOMIC { side } // polyatomic "Got it"
WRONG_ANSWER { side } // inc wrongCount → hint at 2
